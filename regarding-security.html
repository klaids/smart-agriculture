    Node-RED Security

    Following the Securing Node-RED
    you can project your Node-RED dashboard with a password, but you need to change some paramaters on settings.js file on Node-RED directory.
    To do that you need to open the Node-RED directory on your device, is the same if a Raspberry Pi or your Windows notebook, in the first one, you need to show hide files.

    Node-RED folder on Windows OS

    You need to uncomment adminAuth as you can see in this example:

    settings.js example

    Now you need to create your personal password, what you see on the previous image in a hashed password.
    To hash your password open a CLI on a new window:

    node-red admin hash-pw

    Tap the password that you want to use and press Enter, it stamps the hashed password that you can now copy on the file.
    You can create multiple users with different permission, for example an admin user with R/W permission and a second user only with Read Permission, so the last one can't change any part of the Node-RED part.

    External Access

    You can also access from your end device (like smartphone, PC, etc ..) to Node-RED Dashboard.
    To do that we install ngrok on our RPi 3 and tested its service with the Free Plan, ngrok works like a VPN, it creates a encrypted tunnelling connection of the service that you like from the "gateway" (e.g. RPi3) to your end device.
    After installed it, from CLI windows if you put:

    ngrok http https://localhost:1880
    It creates two links, one in http mode for a not encrypted connection, second in https mode, if you copy this link on your browser, you can access to Node-RED.
    If you want secure your connection over https, ngrok allow to create a password to establishing the connection, but this option is available only on Paid Plan.
    Another option is to choose the address to connect with your desidered service, but also in this case you need the Paid Plan.
